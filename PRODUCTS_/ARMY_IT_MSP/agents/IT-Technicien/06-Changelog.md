# IT-Technicien — Changelog

## Conventions de versioning
- **v1.0** : première version opérationnelle (profil + prompt stable + exemples).
- **v1.x** : ajustements compatibles (formulation, sections, nouvelles variantes, amorces).
- **v2.0** : refonte majeure (format de sortie, modes, intégration).

### Quand incrémenter ?
- **Prompt interne modifié** (règles, format sortie, modes) → au minimum **v1.x** (ou v2.0 si rupture).
- **Docs seulement** (amorces/exemples) → v1.x.
- **Changements de périmètre** (exclusions/escalade) → v1.x (ou v2.0 si majeur).
