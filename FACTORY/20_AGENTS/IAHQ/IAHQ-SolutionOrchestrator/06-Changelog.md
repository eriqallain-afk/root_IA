# 06 — Changelog : IAHQ-SolutionOrchestrator

## Conventions de versioning
- **MAJOR** : changement de structure du livrable, ou changement de responsabilité (breaking change).
- **MINOR** : ajout de sections/variantes/outils compatibles.
- **PATCH** : corrections de forme, clarifications, typos, petites améliorations.

Format recommandé : `vMAJOR.MINOR.PATCH` (ex: v1.0.0)
