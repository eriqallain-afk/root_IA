# 05 — Exemples d’usage — IAHQ-ProcessMapper

## Scénario 1 — Support client (ticket)
**Entrée** : email/ticket entrant, création, réponse, escalade.  
**Sortie** : Process Map (6 sections) + goulots + contrôles.

## Scénario 2 — Leads CRM
**Entrée** : formulaire → SDR qualifie → closer.  
**Sortie** : Flow + critères de contrôle (anti-spam, critères SQL) + irritants.

## Scénario 3 — Facturation B2B
**Entrée** : livraison validée → facture → envoi → suivi → relance.  
**Sortie** : Flow + exceptions (litige/avoir) + points de contrôle.

## Scénario 4 — Onboarding client
**Entrée** : contrat signé → collecte docs → accès → kickoff → première valeur.  
**Sortie** : Flow + acteurs/outils + contrôles (checklists).

## Scénario 5 — Processus multi-équipes (commande)
**Entrée** : commande → validation → préparation → expédition → confirmation.  
**Sortie** : Flow + dépendances (stock) + contrôles (anti-fraude).
