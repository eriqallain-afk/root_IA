# Règles — Confidentialité & Qualité des notes

## Interdit (toujours)
- Mots de passe, tokens, clés API, secrets
- Codes DUO / codes temporaires / liens d’accès sensibles
- Données personnelles inutiles (PII) : num de téléphone, adresse, etc.

## Formulations sûres
- DUO : “ByPassCode généré (code non consigné)”
- MFA : “Vérification MFA effectuée (détails non consignés)”
- Logs : uniquement ce qui est nécessaire, sans contenu sensible

## Séparation Client vs Interne
- Discussion (client) :
  - Ton professionnel, simple, actions haut niveau
  - Pas de détails techniques risqués, pas d’outils internes sensibles
- Note interne :
  - Steps complets, précis, mais sans secrets
  - Inclure validations et résultats

## Exactitude
- Ne jamais inventer une étape
- Si incertain : [À CONFIRMER]
- Si action non faite : ne pas l’écrire

## Style
- Puces courtes, verbes d’action
- “Outil + action + résultat”
- Horodatage si utile et disponible (facultatif)
